CONTEXT:
This is a Next.js + Electron desktop app. Characters have already 
been generated and approved. The user now presses 
"Generate Fragmented Script" to parse the story into cinematic 
frames. Each frame will later become one 6-second video clip.

GEMINI SYSTEM PROMPT (use this exact string as the system prompt):
----------------------------------------
export const FRAGMENTS_GENERATION_PROMPT = `
Parse the story into cinematic frames using these rules:

READING THE STORY:
- Read the full story and the approved characters JSON together
- Each frame represents exactly 6 seconds of screen time
- Calculate the number of frames by estimating the story's total 
  video length then dividing by 6:
    1 minute story  = ~10 frames
    2 minute story  = ~20 frames
    3 minute story  = ~30 frames
- Never create fewer than 5 frames regardless of story length
- Each frame must be a single continuous scene — do not split 
  one scene across two frames
- Every meaningful story beat must appear in at least one frame
- Do not skip any part of the story

PER FRAME RULES:
- story_chunk: the exact narrative moment of this frame, 
  2-3 sentences maximum, written in present tense
- scene_description: a detailed visual description of what the 
  camera sees — location, environment, time of day, mood, 
  background details. Specific enough for a video director.
- characters_present: array of character ids from the approved 
  characters JSON — only include characters actually visible 
  in this frame. Use exact ids from characters.json, no new ids.
- camera_hint: one of these exact values only —
  "wide shot" | "medium shot" | "close-up" | "overhead" | 
  "low angle" | "tracking shot"
- duration_seconds: always exactly 6, no exceptions
- transition: one of these exact values only —
  "cut" | "fade" | "dissolve"
  Use "fade" only for the very first and very last frame.
  Use "dissolve" for time passing or location change.
  Use "cut" for everything else.
- status must always be "pending"
- video_path must always be null

CONSISTENCY RULES:
- characters_present ids must exactly match ids in characters.json
  Do not invent new character ids
- scene_description must be consistent with the story's visual style
  inferred from characters.json meta.story_style
- Do not repeat the same camera_hint more than 3 times in a row
- Vary camera_hint to create natural cinematic rhythm

OUTPUT RULES:
- Return ONLY a raw JSON object matching the schema below
- No markdown, no code fences, no explanation, no preamble
- Every string field must be filled — nothing left empty
- frame_id starts at 1 and increments by 1, no gaps
- duration_seconds is always 6
- status is always "pending"
- video_path is always null
- meta.total_frames must equal the actual number of frames generated
- meta.estimated_duration_seconds must equal total_frames × 6

JSON SCHEMA TO FOLLOW:
{
  "meta": {
    "schema_version": "1.0",
    "generated_at": "",
    "total_frames": 0,
    "estimated_duration_seconds": 0,
    "story_style": "",
    "approved": false,
    "locked": false
  },
  "frames": [
    {
      "frame_id": 1,
      "story_chunk": "",
      "scene_description": "",
      "characters_present": [],
      "camera_hint": "",
      "duration_seconds": 6,
      "transition": "",
      "status": "pending",
      "video_path": null
    }
  ]
}
`
----------------------------------------

IMPLEMENTATION:
- Place the prompt constant in lib/prompts/fragments.ts
- This section is only enabled after characters.json has 
  approved: true and locked: true
- Before calling Gemini, load characters.json from:
  userData/projects/{session_id}/skills/characters.json
- Send both to Gemini:
    system prompt = FRAGMENTS_GENERATION_PROMPT
    user message  = "Story:\n{storyText}\n\nApproved Characters:\n{characters JSON}"
- Wrap the entire Gemini call in try/catch
- Before JSON.parse(), strip any accidental markdown fences:
    response.replace(/```json|```/g, '').trim()
- On parse failure: show inline error inside the fragments panel
  with a "Retry" button. Do not crash the app.
- On success:
    - Fill meta.total_frames = frames.length
    - Fill meta.estimated_duration_seconds = frames.length × 6
    - Fill meta.generated_at = new Date().toISOString()
    - Set approved: false, locked: false
    - Save to:
      userData/projects/{session_id}/skills/fragments.json

userData/
